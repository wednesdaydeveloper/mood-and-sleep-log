## 1. 感情タグ追加

### 追加する 11 タグ

| カテゴリ | 既存数 → 新数 | 追加タグ |
|---------|-------------|----------|
| ネガティブ感情 | 10 → 12 | 退屈、物足りない |
| ポジティブ感情 | 9 → 10 | 楽 |
| 状態 | 7 → 15 | 憂鬱、集中できない、軽躁、軽鬱、躁、フラット、アイデアが駆け巡る、過活動 |
| 行動・思考 | 4 | （変更なし） |
| **合計** | **30 → 41** | 11 タグ |

### 設計判断
- **「不安」は重複追加しない**: 既存にあるタグは追加しない（要件確定済み）
- **躁鬱系の医療用語は「状態」カテゴリへ**: 軽躁・軽鬱・躁・憂鬱・フラット・過活動 は感情ではなく状態として扱う
- **「楽」は「ポジティブ感情」**: 「楽しい」と区別する単純な「楽（らく）」の気持ち

### 影響範囲
- `src/domain/tags.ts` — 各カテゴリ配列に追記
- `tests/unit/tags.test.ts` — カテゴリ別件数アサーション、合計 41 アサーション、新規タグ存在確認

### 互換性
- 既存記録のタグは変わらず、新規追加分は次回入力以降に選択可能になるだけ
- DB スキーマ変更なし（`moodTags` は JSON 配列文字列のまま）

## 2. グラフ Y 軸グリッド + ラベル明示

### ゴール
グラフ画面の **気分** / **睡眠時間** の折れ線グラフに、Y 軸ラベル（数値）と水平グリッド罫線を明示し、値を読み取りやすくする。

### 現状
- `MoodChart.tsx` / `SleepDurationChart.tsx` は Victory Native XL の `axisOptions` で `formatYLabel` のみ指定
- グリッド罫線は描画されない、Y 軸ラベル領域が狭く読み取りづらい

### 実装方針
- Y 軸ラベル領域を確保（`padding.left`）
- Y 軸目盛りごとに **水平グリッド罫線**を Skia の `Line` で描画（`renderOutside` 経由）
- ラベル色とフォントサイズの調整

### 影響範囲
- `src/components/chart/MoodChart.tsx`
- `src/components/chart/SleepDurationChart.tsx`
- 共通ユーティリティとして `src/components/chart/HorizontalGridLines.tsx`（任意）

### Y 軸ティック
- 気分: -2, -1, 0, +1, +2（5 段）
- 睡眠時間: データに応じて 4 段（自動レンジ + 1h 刻み）

## 3. リスト検索（キーワード + タグ）

### ゴール
ホーム画面のリストタブに検索バーを常設し、キーワード検索（メモ + タグ名 OR 検索）とタグチップ検索（複数選択 AND）を提供する。

### UI

```
┌────────────────────────────────┐
│ [リスト] [カレンダー]            │ ← 既存
├────────────────────────────────┤
│ 🔍 [キーワード...        ]  ✕  │ ← 検索バー（常設）
│ [+ タグで絞り込み ▾]           │ ← タップで下に展開
├────────────────────────────────┤
│ (リスト本体)                    │
└────────────────────────────────┘
```

タグ展開時:
```
│ ネガティブ感情: 不安  怒り  ...  │
│ ポジティブ感情: 楽しい  嬉しい  │
│ 状態:        鬱  ストレス  ...  │
│ 行動・思考:    「逃げたい」  ... │
```

### 検索ロジック（純粋関数）

```typescript
// src/domain/record-filter.ts
interface FilterCriteria {
  keyword: string;          // 空文字列で無条件マッチ
  selectedTags: string[];   // 空配列で無条件マッチ
}

function matches(record, criteria): boolean {
  // キーワード: メモ OR タグ名 のいずれかにヒット
  const keywordMatch = !criteria.keyword
    || record.memo?.includes(criteria.keyword)
    || record.moodTags.some(t => t.includes(criteria.keyword));

  // タグ: 選択された全部を含む（AND）
  const tagsMatch = criteria.selectedTags.every(t => record.moodTags.includes(t));

  return keywordMatch && tagsMatch;
}

function filterRecords(records, criteria): records[];
```

### 状態管理（Jotai）
```typescript
const searchKeywordAtom = atom('');
const selectedTagsAtom = atom<string[]>([]);
const filteredRecordsAtom = atom((get) => filterRecords(get(recordsAtom), {
  keyword: get(searchKeywordAtom),
  selectedTags: get(selectedTagsAtom),
}));
```

### 影響範囲
- `src/domain/record-filter.ts` 新規 + 単体テスト（10〜15 ケース）
- `src/components/home/HomeListView.tsx` — 検索バー追加
- `src/components/home/SearchBar.tsx` 新規
- `src/components/home/TagFilterChips.tsx` 新規（折りたたみ式）
- `app/(tabs)/index.tsx` — 検索状態をマウント時にリセット

## 4. CSV インポート（既存データ全削除＋取込）

### ゴール
v1.0 でエクスポートした CSV を読み込み、**既存記録を全削除した上で** CSV の内容に置き換える。

### 重要：破壊的操作の安全装置
**2 段階確認ダイアログ** で誤操作を防ぐ:

```
[1段階目]
┌──────────────────────────────────┐
│ ⚠️ 既存データの全削除             │
│                                  │
│ インポートを実行すると、現在の    │
│ すべての記録が削除され、CSV の    │
│ 内容で置き換えられます。          │
│                                  │
│ 元に戻すことはできません。         │
│                                  │
│  [キャンセル]   [次へ]            │
└──────────────────────────────────┘

[2段階目]
┌──────────────────────────────────┐
│ 本当によろしいですか？            │
│                                  │
│ XX 件の記録を削除し、YY 件を      │
│ CSV から取り込みます。            │
│                                  │
│  [キャンセル]   [実行]            │
└──────────────────────────────────┘
```

### 実装方針

#### CSV パース（純粋関数）
```typescript
// src/domain/csv.ts に parseCsv を追加
interface ParsedRecord {
  date: string;
  moodScore: MoodScore;
  moodTags: string[];
  memo: string | null;
  intervals: { startMin: number; endMin: number }[];
}

interface ParseResult {
  records: ParsedRecord[];
  errors: { line: number; message: string }[];
}

function parseCsv(content: string): ParseResult;
```

- UTF-8 BOM 自動除去
- RFC 4180 互換のクォート処理
- 列順は固定（`date,moodScore,moodTags,memo,sleepIntervals`）
- 不正行はエラーリストに追加してスキップ
- 全行不正なら `records: []` を返す

#### Repository 拡張
```typescript
// src/db/repositories/daily-record.ts
async function replaceAll(records: SaveRecordInput[]): Promise<void>;
```
- トランザクション内で `daily_record` を全削除（CASCADE で intervals も削除）
- 一括 `INSERT`
- 失敗時はロールバック

#### 設定画面
- 「📤 CSV エクスポート」の下に「📥 CSV インポート」ボタン
- タップで `expo-document-picker` を起動 → CSV 選択
- パース実行 → 件数表示 → 2 段階確認 → 実行

### 依存追加
- `expo-document-picker`

### エラー表示
- パース失敗（全行不正）: 「CSV の形式が正しくありません」
- 一部不正: 「YY 件取り込み、ZZ 件は形式不正でスキップ」
- DB エラー: 「インポートに失敗しました（ロールバック済み）」

### CSV 形式互換性
- v1.0 でエクスポートした CSV をそのままインポート可能
- 列の追加（v1.x で moodTags が `分類:タグ名` 等になる場合）は将来検討
