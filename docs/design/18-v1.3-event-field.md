# v1.3 イベント項目

> 記録入力画面のメモ欄の **直前** に「イベント（任意）」テキストフィールドを追加し、外出先・予定などの短文を構造化して保存・検索できるようにする。

## 1. 機能要件

### 1.1 入力 UI（記録入力画面 §FR-1.7）

- **メモ欄の直前**に「📅 イベント（任意）」セクションを追加
- 単一行 TextInput
- 文字数上限: **200 文字**
- プレースホルダ: `梅田でショッピング`
- 空欄入力は内部的に `null` に正規化

### 1.2 永続化

- `daily_record` テーブルに 1 カラムを追加
  - `event TEXT`（nullable）
- 既存レコードはマイグレーション直後すべて `null`

### 1.3 検索

- ホームリストのキーワード検索の対象に **event を追加**
- 既存対象: memo / 感情タグ
- 新対象: memo / 感情タグ / **event** （OR 条件、大文字小文字区別なし）

### 1.4 リスト表示

- ホームリストの記録行に event 行を追加
- レイアウト: 日付行 → タグ行 → **event 行** → memo 行
- 表示形式: `📅 {event}`（先頭に絵文字を付けて視認性確保）
- 検索キーワードのハイライト対応

### 1.5 グラフ

- データポイントタップ時のポップアップに「イベント: {event}」行を追加
- week / month のみ（year は集約のため null 固定）
- event が null の日は行を表示しない

### 1.6 CSV

- エクスポート列に **`event`** を追加（8 列形式）
- インポート時の互換性:
  - **v1.3** (8 列) → そのまま読込
  - **v1.2** (7 列、event なし) → event を null として読込
  - **v1.0/v1.1** (5 列、服薬・event なし) → 全て null として読込

## 2. スコープ外（v1.3 では実装しない）

- カレンダービューでのイベント表示（カレンダーセルは現状の絵文字+睡眠時間のまま）
- 過去データの自動移行（Excel コンバータ経由で取り込んだ memo 内の `（イベント：xxx）` 文字列を event 列に抽出する処理）
- イベントによる絞り込みフィルタチップ（キーワード検索のみ）
- イベントの履歴サジェスト・自動補完


`DEFAULT_FORM_VALUES.event = null`。

### 3.3 DB スキーマ（Drizzle）

```typescript

  // 既存カラム ...
```

## 5. Repository / 永続化

### 5.1 toDomain

- `raw.event` をそのまま返す（型ガード不要、長さ制約はフォーム層で担保）

### 5.2 upsert / replaceAll

- 既存の INSERT / UPDATE 文に event カラムを追加

## 6. CSV 連携

### 6.1 エクスポート（recordsToCsv）

- ヘッダー末尾に `event` を追加
- 値は文字列、null は空文字
- 内部の `,` `"` `改行` は標準的にエスケープ（既存の quote 関数）

### 6.2 インポート（parseCsv）

- ヘッダー検出を 3 種に分岐:
  - `current` (8 列、event あり)
  - `legacy_v12` (7 列、v1.2 形式)
  - `legacy_v11` (5 列、v1.0/v1.1 形式)
- 各形式で `expectedColumns` を切り替えて parseRow を実行
- 旧形式では event 欄なし → null

### 6.3 後方互換性

- 旧形式 CSV（5 列・7 列）→ event は null
- 新形式 CSV（8 列）→ そのまま読み込み

## 7. Excel 変換ツール（convert/convert.py）

- v1.2 までは memo 末尾に `（イベント：xxx）` を付加していた
- v1.3 から **専用 event 列に出力**して memo は素のまま
- 既に取り込み済みのデータの memo 内 `（イベント：xxx）` は放置（移行コードなし）

## 10. リスク・注意

| リスク | 対応 |
|--------|------|
| マイグレーション失敗で既存データ破損 | `ADD COLUMN` は追加のみ、既存データに影響しない |
| 旧形式 CSV のインポート失敗 | ヘッダー検出を 3 種に分岐させ、列数も動的に切替 |
| event が長い場合のリスト崩れ | `numberOfLines={1}` + `ellipsizeMode="tail"` で省略 |
| 既存の memo 末尾 `（イベント：xxx）` が冗長 | 放置方針（仕様）。将来再エクスポート → 再インポートで自然に解消 |
