# v1.2 服薬記録機能

> 記録入力画面に「睡眠導入剤」「頓服薬」のラジオボタンを追加し、選択値を記録に保存できるようにする。

## 1. 機能要件

### 1.1 入力 UI（記録入力画面 §FR-1.6）

- メモ欄の **下** に 2 つのセクションを追加
  - **睡眠導入剤**（ラジオボタン、9 択）
  - **頓服薬**（ラジオボタン、4 択）
- いずれも単一選択、デフォルトは「**なし**」
- 「なし」を選択した状態が初期値

### 1.2 選択肢

#### 睡眠導入剤（9 択）
| 値 | 表示 |
|----|------|
| `null` | なし（既定） |
| `lunesta-0.25` | ルネスタ 0.25mg |
| `lunesta-0.5` | ルネスタ 0.5mg |
| `lunesta-0.75` | ルネスタ 0.75mg |
| `lunesta-1.0` | ルネスタ 1.0mg |
| `lunesta-1.5` | ルネスタ 1.5mg |
| `lunesta-2.0` | ルネスタ 2.0mg |
| `lunesta-2.5` | ルネスタ 2.5mg |
| `lunesta-3.0` | ルネスタ 3.0mg |

#### 頓服薬（4 択）
| 値 | 表示 |
|----|------|
| `null` | なし（既定） |
| `lunesta-1.0` | ルネスタ 1.0mg |
| `lunesta-2.0` | ルネスタ 2.0mg |
| `lunesta-3.0` | ルネスタ 3.0mg |

> 内部値は安定キーとして英数字を使う（CSV やマイグレーションでハマらないため）。表示文字列は別マップ `MEDICATION_LABEL` に持たせる。

### 1.3 永続化
- `daily_record` テーブルに 2 カラムを追加
  - `sleep_aid TEXT`（nullable）
  - `prn_medication TEXT`（nullable）
- 既存レコードはマイグレーション直後すべて `null`（= 「なし」）として扱う
- 上記の安定キーをそのまま格納（例 `lunesta-0.5`）

### 1.4 CSV
- エクスポート列に **`sleepAid`** と **`prnMedication`** を追加
  - 値は安定キー、`null` の場合は空文字
- インポート時も同名の列を読み取り、未知のキーや列欠落時は `null` にフォールバック
- v1.0/v1.1 でエクスポートした CSV（列なし）も引き続きインポート可能（後方互換）

## 2. スコープ外（v1.2 では実装しない）

- 検索/絞り込みへの反映（記録のみで十分）
- グラフへの可視化（薬を飲んだ日のマーカー等）
- リスト表示への反映（リストは現状の 2-3 行表示を維持）
- 薬リストのユーザーカスタマイズ（コードでハードコード）
- 服用時刻や服用量メモ（単一選択肢に集約）
- ルネスタ以外の薬（v1.x で必要になれば追加）

## 3. データモデル

### 3.1 ドメイン層
```typescript
// src/domain/medication.ts
export const SLEEP_AID_OPTIONS = [
  'lunesta-0.25',
  'lunesta-0.5',
  'lunesta-0.75',
  'lunesta-1.0',
  'lunesta-1.5',
  'lunesta-2.0',
  'lunesta-2.5',
  'lunesta-3.0',
] as const;
export type SleepAidValue = (typeof SLEEP_AID_OPTIONS)[number];
export type SleepAid = SleepAidValue | null;

export const PRN_MEDICATION_OPTIONS = [
  'lunesta-1.0',
  'lunesta-2.0',
  'lunesta-3.0',
] as const;
export type PrnMedicationValue = (typeof PRN_MEDICATION_OPTIONS)[number];
export type PrnMedication = PrnMedicationValue | null;

export const MEDICATION_LABEL: Record<string, string> = {
  'lunesta-0.25': 'ルネスタ 0.25mg',
  'lunesta-0.5': 'ルネスタ 0.5mg',
  // ...
};

export function isSleepAid(v: unknown): v is SleepAid;
export function isPrnMedication(v: unknown): v is PrnMedication;
```

### 3.2 DB スキーマ（Drizzle）
```typescript
// src/db/schema.ts に追加
export const dailyRecord = sqliteTable('daily_record', {
  // 既存カラム ...
  sleepAid: text('sleep_aid'),                    // nullable
  prnMedication: text('prn_medication'),          // nullable
  // ...
});
```

### 3.3 マイグレーション
- `drizzle-kit generate` で `0001_xxx.sql` を生成
- `migrations.ts` に SQL を template literal で追記（既存運用）
- DDL（おおよそ）:
  ```sql
  ALTER TABLE daily_record ADD COLUMN sleep_aid TEXT;
  ALTER TABLE daily_record ADD COLUMN prn_medication TEXT;
  ```
- 既存レコードは追加カラムが NULL になる（= デフォルト「なし」）

## 6. CSV 連携

### 6.1 エクスポート（recordsToCsv）
- ヘッダーに `sleepAid,prnMedication` を末尾追加
- 値は安定キー、null は空文字
- 例: `2026-04-30,1,"楽しい","良い1日","23:30-07:30","lunesta-0.5",""`

### 6.2 インポート（parseCsv）
- ヘッダーチェックに `sleepAid` / `prnMedication` を含めるが、**両者とも欠落していてもパースを続行**（後方互換）
- 値が空文字または未知のキーは `null` にフォールバック（警告ログ）
- v1.0 / v1.1 でエクスポートした CSV をそのままインポート可能

### 6.3 後方互換性
- 旧形式 CSV（5 列）→ sleepAid/prnMedication は null
- 新形式 CSV（7 列）→ そのまま読み込み

## 9. リスク・注意

| リスク | 対応 |
|--------|------|
| マイグレーション失敗で既存データ破損 | `ADD COLUMN` は追加のみ、既存データに影響しない |
| 旧形式 CSV のインポート失敗 | ヘッダー検証を **必須列のみ**にし、追加列は optional |
| ハードコード値の変更が将来必要 | 安定キー＋ラベルマップで分離。値の renaming はマイグレーション必要 |
| プライバシー（医療情報） | 既存方針（端末ローカルのみ）を維持。CSV 共有時の警告は既存メッセージで十分 |
