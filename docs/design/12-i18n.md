# 言語化・ローカル対応 (i18n)

## 1. スコープ判断

### 現時点
- **対象言語: 日本語のみ**（自分専用、UIテキスト・データとも日本語前提）
- ただし、将来の英語追加に備えた構造を採用する

### 理由
- 感情タグ（不安、楽しい等）は日本語で確定済み
- ストア公開予定なし
- とはいえ、ハードコード文字列を散らすと将来的に拡張が困難

## 2. ライブラリ

```bash
npm install i18next react-i18next expo-localization
```

`react-i18next` を採用（業界標準、Expo 互換、TypeScript 型生成可能）。

## 5. 実装パターン

### 初期化

```typescript
// src/i18n/index.ts
import i18n from "i18next";
import { initReactI18next } from "react-i18next";
import * as Localization from "expo-localization";
import ja from "./locales/ja.json";
import en from "./locales/en.json";

i18n
  .use(initReactI18next)
  .init({
    resources: { ja: { translation: ja }, en: { translation: en } },
    lng: Localization.locale.split("-")[0],
    fallbackLng: "ja",
    interpolation: { escapeValue: false },
  });
```

### 使用

```tsx
import { useTranslation } from "react-i18next";

function HomeHeader() {
  const { t } = useTranslation();
  return <Text>{t("home.title")}</Text>;
}

// 補間
<Text>{t("record.totalSleep", { hours: 7, minutes: 20 })}</Text>
```

### 型安全（将来）
`i18next-typescript` 等で `t("home.title")` のキー補完を有効化。

## 6. 日付・時刻のローカライズ

`date-fns/locale/ja` を使用:

```typescript
import { format } from "date-fns";
import { ja } from "date-fns/locale";

format(new Date(), "yyyy年M月d日(E)", { locale: ja })
// → 2026年4月30日(火)
```

時刻フォーマット（24時間制）はロケール非依存で固定。

## 7. テキストの方針

- 句読点: 全角（、。）
- 数字: 半角
- 単位: 「時間」「分」（24時間制では「:」区切り）
- ボタン: 動詞単体（保存、削除）が基本、否定的な操作は「キャンセル」「破棄」
- エラーメッセージ: 「〜できませんでした」または「〜してください」
- カジュアル度: 敬体（〜です、〜ます）

## 8. 将来の英語化に向けた注意

実装段階で意識:
- **UI内に文字列を直接書かない**（必ず `t()` を経由）
- **数値+単位は補間で扱う**（`"{{count}} hours"` 形式）
- **複数形対応**（i18next の `count` で `_one`, `_other` 自動切替）
- **テキスト長の余裕**（英語は日本語より長くなることも、レイアウトに余白を）

## 9. 当面の進め方

- M1 で i18next を導入、`ja.json` のみ整備
- 全UIテキストを `t()` 経由で書く（生文字列を残さない）
- `en.json` は **作らない**（YAGNI）。将来必要になったときに追加
