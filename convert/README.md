# Excel → CSV 変換ツール

過去に Excel（セルフモニタリング表）で記録していた気分・睡眠データを、
本アプリの v1.4 CSV インポート形式に変換するための **一回限りの移行ツール**。

アプリ本体（`src/`、`app/`）には依存しない、独立した Python スクリプト。

## ディレクトリ構成

```
convert/
├── README.md             ← この文書
├── convert.py            ← 変換スクリプト（管理対象）
├── verify-import.ts      ← 生成 CSV をアプリ parser でスモーク検証（管理対象）
├── {YEAR}セルフモニタリング.xlsx  ← 個人データ（gitignore で除外）
└── converted.csv         ← 生成物（gitignore で除外）
```

## 入力 Excel の前提

ファイル名: `{YEAR}セルフモニタリング.xlsx`（年は先頭 4 文字、例 `2026セルフモニタリング.xlsx`）

シート構成:

- 「空のシート」: 自動スキップ
- **数値月名のシート（例 `1`, `2` ... `12`）**: 各月のデータ
- 月単位以外のシート（例 古い `10` シートのような時間列なしの構造）: スキップ対象

データ行の列構成（ヘッダーは 1 行目）:

| 列名 | 役割 | 備考 |
|------|------|------|
| 日付 | `MM/DD(曜)` 形式 | スクリプトはファイル名の年と組み合わせて `YYYY-MM-DD` を生成 |
| 23, 0, 1, ..., 8 (数値の時間ヘッダー) | 1 時間ごとの睡眠バケット | **セルの塗りつぶし色 `FFFF0000`（赤）が「睡眠中」を意味する**。値そのものは見ない |
| 気分 | -2 〜 +2 | 数値・`+1`・`＋1`（全角）・小数 OK。四捨五入して `[-2, +2]` にクランプ |
| 眠剤 | 数値 (mg) または `無し` | `1.25` は `1.0` に丸め。アプリの enum (`0.25 / 0.5 / 0.75 / 1.0 / 1.5 / 2.0 / 2.5 / 3.0`) に該当しなければ null |
| 頓服 | 数値・`無し`・`有り` | 詳細は下記 |
| 備考 | 自由記述 | memo にそのまま反映 |
| イベント | 任意（200 文字以内） | v1.3 から **専用 event 列**に出力（v1.2 までは memo 末尾に付加していたが分離） |
| 日記 | 任意（5000 文字以内、改行可） | v1.4 から **専用 diary 列**に出力（Dropbox URL 等の長文も格納） |

時間バケットは月によって列数が異なってもよい（例 `22, 23, 0..9` の 12 列も自動検出）。

### キャリーオーバー対応

シート名（月）と日付の月が一致しない行は除外する。
例: 11月シートの末尾に `12/01(月)` の行があっても、12月シートの 12/01 と重複しないよう自動的にスキップ。

### 頓服値のマッピング

| 入力 | 出力 |
|------|------|
| 空 / `無し` / `0` | 空欄（null = 「なし」） |
| `有り` | `lunesta-1.0` |
| 非ゼロ数値 N | `lunesta-{N}` |
| その他のテキスト | 空欄 |

注: アプリの頓服 enum は `1.0 / 2.0 / 3.0` のみ。それ以外（`1.5` 等）は CSV に出力されるが、
取込時に `coercePrnMedication` が null フォールバック（警告ログを出す）。

## 実行手順

### 前提
- Python 3.9+
- `openpyxl`（`pip install openpyxl`）
- Node.js + `npx tsx`（検証用）

### 変換

```bash
# 1. 個人の Excel ファイルを convert/ 配下に置く
#    ファイル名は {YEAR}セルフモニタリング.xlsx 形式

# 2. 変換実行
python convert/convert.py

# → convert/converted.csv が生成される
```

stdout に各シートの処理行数・スキップ件数が出る:

```
読み込み: 2026セルフモニタリング.xlsx
  シート '5': 2 行 (空行スキップ 9)
  シート '4': 30 行 (空行スキップ 0)
  ...
出力: convert/converted.csv (183 行)
```

### 取込前の検証（任意）

アプリの本物の `parseCsv` を通して、取込時にエラーが出ないかを事前確認できる:

```bash
npx tsx convert/verify-import.ts
```

統計と警告ログが表示される（パースエラー 0 件なら OK）。

### アプリへの取込

1. 生成された `convert/converted.csv` を端末に転送（AirDrop / iCloud Drive / メール添付など）
2. アプリ → 設定 → CSV インポート → ファイル選択
3. **既存データはすべて削除されて置き換わる**ため、必要なら事前に CSV エクスポートでバックアップを取る

## 既知の制限

- 時間バケットは 1 時間粒度（Excel の制約）。アプリの 10 分粒度には合わせず、
  バケットの開始/終了をそのまま使う（例: `00:00-08:00`）。
- アプリの頓服 enum (`1.0 / 2.0 / 3.0`) に該当しない用量（`0.5 / 1.5` 等）は null になる。
  必要なら取込後に手動で再設定する。
- 構造の異なる古いシート（時間列がないなど）はスキップされる。`SKIP_SHEETS` 定数で制御。

## 配布除外

`convert/*.xlsx` と `convert/converted.csv` は `.gitignore` で除外している（個人の医療情報に該当するため）。
スクリプト (`convert.py`、`verify-import.ts`、`README.md`) のみ管理対象。
